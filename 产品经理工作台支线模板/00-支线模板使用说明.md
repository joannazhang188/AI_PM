# 产品经理工作台支线模板使用说明

- 文档类型：模板规范
- 适用范围：产品经理工作台 7 条支线
- 用途：统一 7 条支线的输出模板、使用方式和保存建议
- 版本：v1.0
- 更新时间：2026-03-22

## 1. 目录定位

这个目录用于存放产品经理工作台第一阶段可直接落地的 7 条支线模板。

这些模板不是通用方法论文档，而是面向扣子空间中的对话流产出而设计，目标是：

- 可被 AI 稳定理解
- 可在多轮对话中持续修改
- 可直接作为最终保存版本
- 可映射到飞书文档或飞书多维表格

## 2. 覆盖范围

当前目录包含以下 7 条支线模板：

1. 需求线
2. PRD 线
3. PRD 评审线
4. Story 线
5. 测试用例线
6. 测试用例评审线
7. 产品功能更新说明线

说明：

- 原型线当前不在第一阶段模板范围内。
- Story 和测试用例同时提供“文档模板语义”和“结构化字段模板”。

## 3. 使用方式

每个模板建议同时用于两层：

### 3.1 作为提示词中的输出约束

在扣子对话流的大模型节点中，要求模型：

- 严格按模板章节或字段输出
- 缺失信息必须显式标记为“待补充”
- 不允许擅自删除模板中的核心字段

### 3.2 作为最终沉淀格式

- 文档型支线：直接保存为飞书文档
- 结构化支线：先按字段输出 JSON，再写入飞书多维表格

## 4. 字段使用原则

- 未确认的信息不要编造，统一标记为“待确认”。
- 每个字段都优先写可判断、可执行、可保存的内容。
- 对 AI 相关内容，必须写清输入、输出、质量要求和失败兜底。
- 评审类模板优先输出问题、风险、建议，不写空泛赞同。

## 5. 推荐保存方式

| 支线 | 推荐保存方式 |
| --- | --- |
| 需求线 | 飞书文档 |
| PRD 线 | 飞书文档 |
| PRD 评审线 | 飞书文档 |
| Story 线 | 飞书多维表格 |
| 测试用例线 | 飞书多维表格 |
| 测试用例评审线 | 飞书文档 |
| 产品功能更新说明线 | 飞书文档 |

## 6. 推荐在扣子中的调用方式

- 需求线、PRD 线、PRD 评审线、测试用例评审线、产品功能更新说明线：
  直接要求输出 Markdown。

- Story 线、测试用例线：
  先要求模型输出结构化 JSON，再通过代码节点转换成飞书多维表格写入格式。

## 7. 版本规则

- 调整模板结构时升级主版本。
- 仅补说明或微调措辞时升级小版本。
- 模板文件名保持稳定，不在文件名中加入日期。
