# 字段规范-测试用例极简版-v1

- 文档类型：字段规范
- 适用阶段：测试用例线
- 用途：定义测试用例的最小可用字段，用于知识库约束、扣子结构化输出和飞书多维表格建表
- 版本：v1.0
- 更新时间：2026-03-22

## 1. 使用原则

- 只保留测试执行和验收最关键的字段。
- 保证 AI 可稳定输出，避免字段过多导致 token 浪费。
- 优先服务飞书多维表格落地。

## 2. 推荐字段

| 字段名               | 类型        | 必填  | 说明          | 填写要求             | 示例                           |
| ----------------- | --------- | --- | ----------- | ---------------- | ---------------------------- |
| `case_id`         | Text      | 是   | 用例唯一编号      | 保持简短且唯一          | `TC-001`                     |
| `project_name`    | Text      | 是   | 所属项目名称      | 使用统一项目名          | `产品经理工作台`                    |
| `module_name`     | Text      | 是   | 所属模块        | 便于筛选和归类          | `需求线`                        |
| `source_story_id` | Text      | 是   | 对应 Story 编号 | 用于追溯来源           | `ST-001`                     |
| `case_title`      | Text      | 是   | 用例标题        | 直接说明测什么          | `上传 PDF 后可生成需求摘要`            |
| `precondition`    | Long Text | 否   | 前置条件        | 账号、权限、环境等        | `已登录；文件解析能力可用`               |
| `test_steps`      | Long Text | 是   | 测试步骤        | 用顺序描述，便于执行       | `1. 进入需求线 2. 上传 PDF 3. 提交问题` |
| `expected_result` | Long Text | 是   | 预期结果        | 必须可判断            | `成功生成摘要，并输出待确认问题`            |
| `priority`        | Enum      | 是   | 优先级         | 建议固定为 `P0/P1/P2` | `P0`                         |
| `remark`          | Long Text | 否   | 备注          | 补充边界说明或特殊样本      | `异常文件另补用例`                   |

## 3. 推荐枚举值

### `priority`

- `P0`
- `P1`
- `P2`

## 4. 推荐 JSON 输出格式

```json
[
  {
    "case_id": "",
    "project_name": "",
    "module_name": "",
    "source_story_id": "",
    "case_title": "",
    "precondition": "",
    "test_steps": "",
    "expected_result": "",
    "priority": "",
    "remark": ""
  }
]
```

## 5. 飞书多维表格建表建议

推荐至少建立以下列：

- 用例编号
- 项目名称
- 所属模块
- 来源 Story
- 用例标题
- 前置条件
- 测试步骤
- 预期结果
- 优先级
- 备注

## 6. 知识库使用建议

把这份文档放进知识库后，测试用例线的大模型节点应明确要求：

- 必须按这 10 个字段输出
- 不允许擅自新增字段
- 缺失信息统一写 `待确认`
