# 模板-PRD线-v1

- 文档类型：模板
- 适用阶段：PRD 线
- 用途：把已确认的需求说明整理成可评审、可执行的 PRD
- 触发条件：需求说明已基本清晰，用户要求生成或修改 PRD
- 上一阶段输入：需求说明 / 补充资料
- 下一阶段输出：Story 线 / PRD 评审线 / 产品功能更新说明线
- 版本：v1.0

## 字段填写说明

- 项目背景：先写为什么做，再写做什么。
- 问题定义：明确本次 PRD 要解决哪一段问题。
- 目标与成功标准：必须可判断，避免空话。
- 范围与边界：一定要同时写本期范围和非本期范围。
- 功能需求：按模块写清用户操作、系统行为、规则和异常。
- AI 能力设计：必须写输入、输出、质量要求和兜底。
- 验收标准：每一条都要可验证。
