# 模板-测试用例线-v1

- 文档类型：模板
- 适用阶段：测试用例线
- 用途：根据 PRD 和 Story 生成可执行、可验证的测试用例
- 触发条件：Story 已确认，需要准备测试设计与验收
- 上一阶段输入：PRD / Story
- 下一阶段输出：测试执行 / 缺陷跟踪 / 上线验收
- 版本：v1.0

## 字段填写说明

- 用例标题：明确写“在什么条件下验证什么结果”。
- 测试目标：一条用例只测一个核心点。
- 前置条件：写账号、权限、环境、配置和依赖资源。
- 测试数据：区分正常、异常、边界样本。
- AI 输出预期：必须写结构和最低可用标准。
- 异常与兜底：AI 场景必须写清。

## 结构化字段模板

测试用例线最终推荐输出为 JSON 数组，字段如下：

```json
[
  {
    "case_id": "",
    "project_name": "",
    "module_name": "",
    "source_story_id": "",
    "case_title": "",
    "precondition": "",
    "test_steps": "",
    "expected_result": "",
    "priority": "",
    "remark": ""
  }
]
```

## 使用提醒

- 如果要直接落飞书多维表格，建议把 `test_steps`、`expected_result` 保持为文本字段，不要拆成过多子列。
- 一条用例只验证一个核心点，避免一条用例覆盖多个目标。
