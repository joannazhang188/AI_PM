# 模板-Story线-v1

- 文档类型：模板
- 适用阶段：Story 线
- 用途：把 PRD 拆成可执行、可分配、可验收的 Story
- 触发条件：PRD 已确认，需要进入执行拆解
- 上一阶段输入：PRD
- 下一阶段输出：测试用例线 / 开发排期 / 任务协作
- 版本：v1.0

## 字段填写说明

- Story 标题：建议使用“动作 + 对象 + 目标”。
- 用户故事：保持“作为……我希望……从而……”格式。
- 范围：同时写包含和不包含。
- AI 能力说明：AI 相关 story 必须写输入、输出和兜底。
- 验收标准：每条都要可验证。

## 结构化字段模板

Story 线最终推荐输出为 JSON 数组，字段如下：

```json
[
  {
    "story_id": "",
    "project_name": "",
    "source_prd": "",
    "story_title": "",
    "user_story": "",
    "task_description": "",
    "acceptance_criteria": "",
    "priority": "",
    "estimate_days": "",
    "dependencies": ""
  }
]
```

## 使用提醒

- 如果最终要写入飞书多维表格，优先使用结构化字段模板。
- 如果用于人工评审或确认，可先输出文档模板版本。
