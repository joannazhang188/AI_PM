# 字段规范-Story极简版-v1

- 文档类型：字段规范
- 适用阶段：Story 线
- 用途：定义 Story 的最小可用字段，用于知识库约束、扣子结构化输出和飞书多维表格建表
- 版本：v1.0
- 更新时间：2026-03-22

## 1. 使用原则

- 只保留推进执行和验收最关键的字段。
- 能同时服务知识库、扣子输出和飞书落表。
- 不保留当前阶段非必要的管理字段。

## 2. 推荐字段

| 字段名                   | 类型          | 必填  | 说明         | 填写要求              | 示例                             |
| --------------------- | ----------- | --- | ---------- | ----------------- | ------------------------------ |
| `story_id`            | Text        | 是   | Story 唯一编号 | 保持简短且唯一           | `ST-001`                       |
| `project_name`        | Text        | 是   | 所属项目名称     | 使用统一项目名           | `产品经理工作台`                      |
| `source_prd`          | Text        | 是   | 来源 PRD     | 填 PRD 名称或版本       | `PRD-V1`                       |
| `story_title`         | Text        | 是   | Story 标题   | 用“动作 + 对象 + 目标”命名 | `支持上传需求资料并生成需求摘要`              |
| `user_story`          | Long Text   | 是   | 用户故事描述     | 使用“作为…我希望…从而…”句式  | `作为产品经理，我希望上传需求资料，从而快速整理需求说明。` |
| `task_description`    | Long Text   | 是   | 任务描述       | 写清主要任务点，避免空话      | `支持文本、文件、图片输入，并输出结构化摘要。`       |
| `acceptance_criteria` | Long Text   | 是   | 验收标准       | 写成可验证条件           | `可上传资料；可生成摘要；缺信息时给出待补充项。`      |
| `priority`            | Enum        | 是   | 优先级        | 建议固定为 `P0/P1/P2`  | `P0`                           |
| `estimate_days`       | Number/Text | 否   | 预估人天       | 没有可留空             | `2`                            |
| `dependencies`        | Long Text   | 否   | 依赖项        | 只写真实依赖            | `依赖文件解析子工作流`                   |

## 3. 推荐枚举值

### `priority`

- `P0`
- `P1`
- `P2`

## 4. 推荐 JSON 输出格式

```json
[
  {
    "story_id": "",
    "project_name": "",
    "source_prd": "",
    "story_title": "",
    "user_story": "",
    "task_description": "",
    "acceptance_criteria": "",
    "priority": "",
    "estimate_days": "",
    "dependencies": ""
  }
]
```

## 5. 飞书多维表格建表建议

推荐至少建立以下列：

- Story 编号
- 项目名称
- 来源 PRD
- Story 标题
- 用户故事
- 任务描述
- 验收标准
- 优先级
- 预估人天
- 依赖项

## 6. 知识库使用建议

把这份文档放进知识库后，Story 线的大模型节点应明确要求：

- 必须按这 10 个字段输出
- 不允许擅自新增字段
- 缺失信息统一写 `待确认`
