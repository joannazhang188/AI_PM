# 产品经理工作台 UI需求文档 / 页面说明书

## 1. 文档目的

本文档面向前端开发，目标是让开发可以按 1:1 复刻设计截图完成工作台页面搭建。

本平台不是多页面网站，而是一个工作台容器下的多视图切换应用。

- 左侧固定为侧边栏
- 右侧为主内容区
- 右侧仅在不同视图之间切换
- 不额外新增独立页面路由

## 2. 页面范围

整个平台只有以下 4 个主视图：

1. 首页欢迎视图
2. 资源库视图
3. 对话视图
4. 对话 + 文档联动视图

这 4 个视图共用同一套外层框架。

## 3. 总体布局

### 3.1 外层结构

页面为左右二栏布局：

- 左侧：侧边栏
- 右侧：主内容区

结构建议：

```text
WorkbenchPage
|- Sidebar
|- MainPanel
   |- HomeView | LibraryView | ConversationView | ConversationDocumentView
```

### 3.2 布局行为

- 左侧侧边栏支持展开和收起
- 右侧主内容区宽度随侧边栏状态自适应
- 侧边栏始终可见，不因右侧视图切换而消失
- 页面整体视觉为浅色、轻拟物、卡片化、圆角风格

## 4. 左侧侧边栏说明

侧边栏为平台公共区域，所有视图共用。

### 4.1 侧边栏区域划分

从上到下分为 4 个区域：

1. 品牌区
2. 一级操作区
3. 无项目对话目录
4. 项目目录区

### 4.2 品牌区

包含：

- Logo 图标
- 标题：`产品经理工作台`
- 侧边栏展开/收起按钮

要求：

- Logo 在展开和收起态都显示
- 展开态显示标题文字
- 收起态不显示标题文字
- 折叠按钮在两种状态都保留

### 4.3 一级操作区

固定包含 3 个按钮：

1. `新建项目`
2. `新对话`
3. `资源库`

要求：

- 展开态显示 icon + 文案
- 收起态仅显示 icon
- 三个按钮顺序固定，不可调整

### 4.4 无项目对话目录

这是固定目录，不是项目。

目录名称：

- `无项目对话`

用途：

- 存放所有不归属任何项目的独立对话

规则：

- 仅当存在独立对话时显示
- 支持展开/收起
- 对话按创建时间倒序排列，最新创建的在最上方

### 4.5 项目目录区

展示所有项目及项目下对话。

项目规则：

- 项目按创建时间排序
- 每个项目为一个目录节点
- 支持展开/收起
- 项目下对话按创建时间倒序排列

项目更多操作包含：

- `新建对话`
- `重命名`
- `删除`

对话更多操作包含：

- `重命名`
- `删除`
- `移动至项目`

### 4.6 侧边栏空状态

当没有项目且没有独立对话时：

- 仅显示品牌区
- 仅显示 3 个一级按钮
- 不显示 `无项目对话` 目录
- 不显示项目目录区

### 4.7 侧边栏收起态

收起态表现要求：

- 保留 Logo
- 保留折叠按钮
- `新建项目`、`新对话`、`资源库` 仅显示 icon
- 不显示 `无项目对话`
- 不显示项目目录区
- 不显示任何目录文案

## 5. 右侧主内容区视图说明

右侧主内容区根据当前状态切换视图。

建议状态：

```ts
type MainView = "home" | "library" | "conversation";
```

文档联动不单独作为页面类型，而是 `conversation` 视图下的扩展布局。

## 6. 首页欢迎视图

### 6.1 用途

这是平台默认进入的视图，用于承接首次输入和快捷能力入口。

### 6.2 页面结构

从上到下包含：

1. 智能体标签
2. 主标题
3. 输入区
4. 快捷入口区

### 6.3 页面内容

顶部标签示例：

- `产品团队超级智能体`

主标题：

- `需要我为你做些什么呢？`

### 6.4 输入区

输入区为大号卡片，包含：

- 多行文本输入框
- 上传文件入口
- 上传图片入口
- `引用资源库`
- 发送按钮

要求：

- 输入区在首页居中展示
- 风格贴近截图中的白色圆角卡片
- 工具按钮贴底排列

### 6.5 快捷入口区

位于输入区下方。

内容为能力卡片网格，至少包括：

- `需求澄清`
- `生成PRD`
- `需求评审`
- `任务拆解`
- `生成测试用例`
- `测试用例评审`
- `产品更新说明`
- `产品竞品分析`
- `产品调研报告`
- `产品介绍 PPT`

要求：

- 每个卡片为圆角胶囊样式
- 包含 icon + 文案
- 支持 hover 态

## 7. 资源库视图

### 7.1 用途

资源库不是独立页面，而是在右侧主内容区切换展示。

### 7.2 进入方式

- 点击左侧 `资源库`

### 7.3 页面结构

从上到下包含：

1. 返回入口
2. 页面标题
3. 资源表格

### 7.4 页面内容

顶部返回按钮：

- `返回对话`

标题：

- `资源库`

表格字段：

- `资源 ID`
- `资源名称`
- `所属类型`
- `创建日期`
- `说明`
- `操作`

每条资源操作包含：

- `引用至对话`
- `删除`

### 7.5 资源库返回规则

- 若进入资源库前处于对话页，点击 `返回对话` 后返回之前的对话页
- 若进入资源库前不在对话页，则返回首页欢迎视图

## 8. 对话视图

### 8.1 用途

点击任一对话后，右侧进入对话视图。

### 8.2 页面结构

对话视图分为上下两层：

1. 上层：消息展示区
2. 下层：固定输入区

上层消息区顶部包含对话头部栏。

### 8.3 对话头部栏

左侧：

- 当前对话标题

右侧：

- `退出对话`

示例标题：

- `准备需求评审`

### 8.4 消息展示区

消息区要求：

- 支持空状态
- 支持已有消息状态
- 支持纵向滚动
- 输入区固定在底部，不与消息一起滚动

### 8.5 消息类型

至少支持以下几类：

1. 用户消息气泡
2. AI 文本回复
3. 资源引用卡片
4. 文档卡片

AI 回复中允许出现：

- 列表
- 步骤说明
- 路径代码块
- 引用卡片

### 8.6 文档卡片

从截图看，对话中可出现文档卡片。

卡片元素至少包含：

- 文档名称
- 操作按钮：`引用`

示例：

- `PRD 精简版文档卡片`

### 8.7 底部输入区

输入区固定在对话底部，包含：

- 多行输入框
- 上传文件按钮
- 上传图片按钮
- `引用资源库`
- 快捷能力选择器
- 语音入口
- 发送按钮

要求：

- 宽度铺满对话容器底部
- 与截图一致，保持独立卡片式容器
- 不随消息滚动

## 9. 对话 + 文档联动视图

### 9.1 用途

当对话生成结构化文档，或用户打开某个文档时，右侧主内容区切换为聊天与文档联动布局。

### 9.2 布局结构

采用左右双栏：

- 左栏：对话区
- 右栏：文档面板

左栏保留对话视图原有内容：

- 顶部对话头部栏
- 消息区
- 底部固定输入区

### 9.3 文档面板结构

文档面板包含：

1. 顶部工具栏
2. 文档内容区

### 9.4 文档工具栏

顶部工具栏包含：

- 收起/返回按钮
- `代码`
- `预览`
- `导出`
- `编辑`
- `复制`

### 9.5 文档内容区

文档内容区应支持：

- 文档标题
- 文档生成时间
- Markdown 标题
- 正文段落
- 长文滚动查看

### 9.6 文档面板规则

- 默认支持 `代码` 和 `预览` 两种模式
- 面板可收起
- 收起后回到普通对话视图
- 文档仅作为当前对话上下文的联动面板，不切换左侧侧边栏

## 10. 业务规则汇总

### 10.1 新建对话

- 在无项目时允许直接创建
- 直接创建的对话归类到 `无项目对话`

### 10.2 项目下新建对话

- 只有从某项目内触发新建对话时，该对话才归属该项目

### 10.3 排序规则

项目：

- 按创建时间排序

对话：

- 按创建时间倒序排序
- 最近创建的在最上面

### 10.4 资源库切换规则

- 点击左侧 `资源库`，右侧切换到资源库视图
- 不跳独立页面

### 10.5 对话退出规则

- 点击 `退出对话`，右侧返回首页欢迎视图
- 不删除当前对话数据

## 11. 建议数据结构

```ts
type ProjectItem = {
  id: string;
  name: string;
  createdAt: string;
};

type ConversationItem = {
  id: string;
  title: string;
  createdAt: string;
  projectId: string | null;
};

type ResourceItem = {
  id: string;
  name: string;
  type: string;
  createdAt: string;
  description: string;
};

type DocumentItem = {
  id: string;
  conversationId: string;
  title: string;
  content: string;
  format: "markdown" | "text";
  createdAt: string;
  updatedAt?: string;
};

type WorkbenchState = {
  sidebarCollapsed: boolean;
  mainView: "home" | "library" | "conversation";
  documentPanelOpen: boolean;

  activeProjectId: string | null;
  activeConversationId: string | null;
  activeDocumentId: string | null;

  expandedProjectIds: string[];
  ungroupedFolderExpanded: boolean;

  lastConversationId: string | null;

  projects: ProjectItem[];
  conversations: ConversationItem[];
  resources: ResourceItem[];
  documents: DocumentItem[];
};
```

## 13. 视图切换规则

开发按以下方式渲染右侧内容：

- `mainView === "home"`：显示 `HomeView`
- `mainView === "library"`：显示 `LibraryView`
- `mainView === "conversation" && documentPanelOpen === false`：显示 `ConversationView`
- `mainView === "conversation" && documentPanelOpen === true`：显示 `ConversationDocumentView`

## 14. 1:1 复刻要求

为保证能复刻截图，开发时必须遵守以下原则：

### 14.1 结构优先

- 不改变左右二栏基础结构
- 不新增顶部全局导航
- 不把资源库改成弹窗或新页面
- 不把文档面板改成全屏页

### 14.2 视觉优先

- 使用浅色背景
- 使用大圆角白色卡片
- 页面留白要充足
- 左右边距、上下分区要接近截图
- 输入区、消息区、文档区都保持独立边界

### 14.3 交互优先

- 侧边栏收起后仅保留 icon 级操作
- 对话输入区固定底部
- 对话消息区可独立滚动
- 文档面板独立滚动
- 右侧视图切换不能影响左侧目录状态

## 15. 开发顺序建议

建议按以下顺序搭建：

1. 创建 `WorkbenchPage` 外层容器
2. 实现 `Sidebar` 展开/收起与空状态
3. 实现 `HomeView`
4. 实现 `LibraryView`
5. 实现 `ConversationView`
6. 实现 `ConversationDocumentView`
7. 最后接入项目、对话、资源、文档真实数据

## 16. 验收标准

完成后应满足以下条件：

1. 平台仅由一个工作台壳层承载全部视图
2. 左侧侧边栏支持展开和收起
3. 空状态下侧边栏仅保留 3 个一级按钮
4. 独立对话统一进入 `无项目对话` 目录
5. 项目下新建对话自动归属项目
6. 点击 `资源库` 时右侧切换资源库视图
7. 点击对话时右侧切换对话视图
8. 对话页支持空状态和已有消息状态
9. 对话页输入区固定底部
10. 对话中支持文档卡片和资源卡片
11. 打开文档后右侧进入聊天 + 文档联动布局
12. 文档面板支持 `代码`、`预览`、`导出`、`编辑`、`复制`
13. 整体页面结构、内容层级和交互状态可 1:1 对齐提供的截图

## 17. 补充说明

- 本文档描述的是 UI 结构与页面行为，不包含后端接口定义
- 若进入开发前需要进一步细化，可在此文档基础上继续补充字段、接口和状态流转图
- 若要完全贴齐截图，开发时应以本说明书为骨架、以截图为最终视觉校对基准
