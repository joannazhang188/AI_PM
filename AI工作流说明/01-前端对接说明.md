# 前端对接说明

## 1. 目标

前端的目标不是生成模型逻辑，而是稳定承接工作流阶段，保证：

- 工作流选择、消息发送、文档查看、文档编辑、引用、导出不会串线
- 输入区状态与请求体一致
- 文档卡片、右侧文档栏、资源库之间使用同一份真实数据

## 2. 前端核心状态模型

### 2.1 输入区草稿

```ts
type ComposerDraft = {
  text: string;
  selectedWorkflowId: string | null;
  attachments: ComposerAttachmentRef[];
  quotedArtifactIds: string[];
  quotedResourceIds: string[];
  modelMode: "auto" | "fast" | "quality";
};
```

说明：

- `selectedWorkflowId` 任一时刻只允许一个
- `quotedArtifactIds` 只存引用的文档 ID
- `quotedResourceIds` 只存引用的资源 ID
- 不要把完整 artifact 或 resource 正文塞进输入区状态

### 2.2 会话状态

```ts
type ConversationState = {
  id: string;
  currentWorkflowId: string | null;
  messages: Message[];
};
```

发送规则：

- 若本轮 `ComposerDraft.selectedWorkflowId` 有值，则覆盖 `ConversationState.currentWorkflowId`
- 若本轮为空，则本轮请求视为普通对话，同时把 `currentWorkflowId` 清空

### 2.3 文档实体

```ts
type ArtifactDocument = {
  id: string;
  conversationId: string;
  sourceMessageId: string;
  workflowId: string | null;
  title: string;
  format: "md" | "html" | "json" | "text";
  content: string;
  createdAt: string;
  updatedAt: string;
  version: number;
  isEdited: boolean;
};
```

前端只维护实体，不维护多份正文副本。

## 3. 前端界面行为规则

## 3.1 工作流选择

- 输入区只允许单选一个快捷功能
- 选中后显示为一个标签
- 更换快捷功能时，原标签立即被替换
- 切回“选择快捷功能”时，表示 `selectedWorkflowId = null`
- 本轮发送一定以当前标签为准

## 3.2 输入区引用

引用类型：

- 上传文件
- 上传图片
- 引用文档 artifact
- 引用资源 resource

展示规则：

- 全部展示为统一的输入区标签
- 标签应能删除
- 若资源被删除，当前输入区里的对应引用应立即移除

## 3.3 文档卡片

### 点击卡片主体

- 打开右侧文档栏
- 展示该 artifact 的详情

### 点击“引用”

- 只把 artifact 引用加入输入区
- 不自动打开右侧文档栏

## 3.4 文档栏

### 预览模式

- 预览内容必须由 `artifact.content` 实时派生
- 不维护静态 preview 副本

### 代码模式

- 展示原始内容

### 编辑模式

- 编辑的是 `artifact.content`
- 保存后回写 artifact
- 预览、复制、引用、导出都读取最新内容

### 复制按钮

- 编辑态：复制当前编辑内容
- 代码态：复制原始内容
- 预览态：复制当前预览文本

## 3.5 模型阶段展示

前端必须根据后端返回的 `stage` 决定展示方式。

### `clarify`

- 展示追问内容
- 不创建正式 artifact

### `template_confirm`

- 展示模板建议或大纲
- 提供“确认模板”或继续修改模板的交互入口

### `draft_generate`

- 渲染普通回复文本
- 同时渲染 artifact 卡片

### `revise_existing`

- 渲染“已基于原文修改”的回复
- 更新原 artifact 实体
- 卡片仍指向原 artifactId

## 4. 前端请求规则

发送消息时应构造统一请求：

```json
{
  "conversationId": "conv_001",
  "messageText": "请基于当前 PRD 修改登录流程",
  "workflowId": "prd",
  "attachments": [],
  "quotedArtifactIds": ["art_001"],
  "quotedResourceIds": ["res_001"],
  "targetArtifactId": "art_001",
  "modelMode": "quality"
}
```

关键点：

- `workflowId` 取当前输入区选中的快捷功能
- `quotedArtifactIds` 和 `quotedResourceIds` 由输入区标签生成
- `targetArtifactId` 仅在明确修改某份文档时携带

## 5. 前端错误与边界处理

### 5.1 失效引用

- 若资源已删除，历史引用卡片显示“该文档已不存在”
- 点击时只提示，不允许再次引用

### 5.2 模板未确认

- 若后端返回 `stage = template_confirm`
- 前端不能把返回内容当最终文档保存到资源库

### 5.3 编辑后的内容优先

若 artifact 已编辑，则：

- 再次打开右栏使用编辑后版本
- 再次引用使用编辑后版本
- 导出资源库保存编辑后版本

## 6. 前端验收要点

- 同一时刻只能存在一个工作流标签
- 清空工作流后，本轮发送按普通对话处理
- 点击文档卡片与点击引用行为分离
- 文档编辑后，预览立即跟随更新
- 删除资源后，当前输入区引用同步失效
- 历史消息中的失效引用有明确视觉态和错误提示
