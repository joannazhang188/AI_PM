# 后端工作流与接口说明

## 1. 目标

后端需要把“快捷功能”从提示词拼接模式升级成工作流执行模式，重点解决：

- 当前到底是普通对话还是专用工作流
- 当前阶段到底是追问、确认模板、生成初稿还是修改原文
- 文档和资源如何成为后续上下文中的真实内容

## 2. 核心模块

推荐后端拆成以下 5 个模块。

## 2.1 `workflowRegistry`

作用：

- 统一维护 11 个快捷功能的定义
- 决定是否要先澄清
- 决定是否需要模板确认
- 决定输出物类型
- 决定是否支持联网
- 决定是否支持基于原文修改

建议结构：

```ts
type WorkflowDefinition = {
  id: string;
  name: string;
  goal: string;
  requiresClarify: boolean;
  requiresTemplateConfirm: boolean;
  supportsRevision: boolean;
  outputType: "document" | "html" | "table";
  outputFormat: "md" | "html" | "json" | "text";
  allowWebSearch: boolean;
};
```

## 2.2 `contextBuilder`

作用：

- 组装模型上下文
- 保证引用的资源和文档正文真正进入模型

输入来源：

- 用户消息文本
- 上传文件文本
- 上传图片理解结果
- 引用 artifact 正文
- 引用 resource 正文
- 历史会话摘要
- 当前 workflow 定义

要求：

- resource 和 artifact 必须读取真实 `content`
- 不允许只把资源标题传给模型

## 2.3 `stageController`

作用：

- 判断当前会话应该进入哪个阶段

判断依据：

- workflow 是否为空
- 素材是否充分
- 用户是否已确认模板
- 是否带了 `targetArtifactId`
- 用户意图是否是“修改现有文档”

## 2.4 `artifactService`

作用：

- 创建 artifact
- 更新 artifact
- 生成版本号
- 提供 artifact 正文给引用和导出流程

规则：

- 编辑保存后更新同一 artifact
- 修改文档时默认更新目标 artifact，而不是创建无关新 artifact

## 2.5 `resourceService`

作用：

- 导出 artifact 到资源库
- 删除资源
- 为引用资源提供正文内容

规则：

- 保存正文
- 保存格式
- 保存来源工作流
- 保存来源会话
- 保存是否为编辑后内容

## 3. 聊天请求协议

建议后端统一使用如下请求体。

```json
{
  "conversationId": "conv_001",
  "messageText": "请继续修改第 3 节",
  "workflowId": "prd",
  "attachments": [
    {
      "id": "att_001",
      "kind": "file",
      "name": "需求摘要.md",
      "payload": {
        "mode": "text",
        "text": "..."
      }
    }
  ],
  "quotedArtifactIds": ["art_001"],
  "quotedResourceIds": ["res_001"],
  "targetArtifactId": "art_001",
  "templateConfirmed": false,
  "templateSource": "default",
  "modelMode": "quality"
}
```

字段说明：

- `workflowId = null` 表示普通对话
- `quotedArtifactIds` 表示引用的文档
- `quotedResourceIds` 表示引用的资源库资源
- `targetArtifactId` 表示本轮是针对某份文档修改
- `templateConfirmed` 表示用户是否已接受模板

建议统一返回：

```json
{
  "assistantText": "以下是建议的 PRD 大纲，请确认。",
  "workflowUsed": "prd",
  "stage": "template_confirm",
  "needsUserConfirmation": true,
  "questions": [],
  "templateProposal": {
    "title": "PRD 默认模板",
    "content": "..."
  },
  "artifacts": [],
  "citations": []
}
```

当生成正式产物时：

```json
{
  "assistantText": "我已根据确认的大纲生成 PRD 初稿。",
  "workflowUsed": "prd",
  "stage": "draft_generate",
  "needsUserConfirmation": false,
  "questions": [],
  "templateProposal": null,
  "artifacts": [
    {
      "id": "art_001",
      "type": "document",
      "format": "md",
      "title": "狼人杀小程序 PRD",
      "content": "..."
    }
  ],
  "citations": []
}
```

## 5. 模板确认规则

后端应统一支持 4 类模板来源：

- `uploaded_template`
- `quoted_template`
- `user_defined_template`
- `system_default_template`

逻辑规则：

- 若用户上传模板且明确要求按此模板生成，可直接进入 `draft_generate`
- 若用户没有模板，则返回默认模板提议，进入 `template_confirm`
- 若用户要求修改模板，则继续停留在 `template_confirm`
- 只有模板确认后才能进入正式成品生成

## 6. 文档修改规则

进入 `revise_existing` 的条件：

- `targetArtifactId` 存在
- 或用户引用了 artifact 且意图明显为“修改”

此时：

- `contextBuilder` 需读取目标 artifact 正文
- 模型输出必须基于原文改写
- `artifactService` 更新同一 artifact 的内容和版本号

不要做成：

- 每次修改都新建一份完全独立的新文档

## 7. 联网能力规则

以下工作流默认允许联网：

- 产品竞品分析
- 产品调研报告

联网结果要求：

- 保留来源链接或来源信息
- 可在 `citations` 中结构化返回
- 前端可选择展示来源列表

建议结构：

```json
{
  "citations": [
    {
      "title": "竞品官网",
      "url": "https://example.com",
      "sourceType": "web"
    }
  ]
}
```

## 8. Artifact 更新接口

建议保留：

```http
PATCH /api/artifacts/:id
```

请求体：

```json
{
  "title": "更新后的 PRD 标题",
  "format": "md",
  "content": "..."
}
```

返回：

```json
{
  "artifact": {
    "id": "art_001",
    "title": "更新后的 PRD 标题",
    "format": "md",
    "content": "...",
    "updatedAt": "2026-03-24 20:00:00",
    "version": 3,
    "isEdited": true
  }
}
```

## 9. 资源库存储接口

建议保留：

```http
POST /api/resources
```

请求体：

```json
{
  "sourceArtifactId": "art_001",
  "workflowId": "prd",
  "workflowName": "生成PRD",
  "name": "狼人杀小程序 PRD",
  "format": "md",
  "content": "...",
  "conversationId": "conv_001",
  "conversationTitle": "狼人杀项目讨论"
}
```

要求：

- 内容来自最新 artifact.content
- 若 artifact 已编辑，资源库保存编辑后版本

## 10. 后端验收要点

- `workflowId = null` 时，不进入专用工作流
- `stage` 返回明确，前端不需要猜
- 文档修改时更新原 artifact
- 资源库引用进入模型上下文的是正文
- 联网工作流返回来源信息
- 模板未确认前不生成正式成品
